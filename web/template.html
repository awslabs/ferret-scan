üîç Ferret Scan Help & Documentation

üåê How to Use the Web Interface

This web interface provides an easy way to scan files for sensitive data without using the command line. Follow these steps to get started:

‚ö° Quick Start

  1. Click "Choose Files" or drag files onto the upload area
  2. Optionally adjust confidence levels and detection types in "Scan Configuration"
  3. Click "Start Scan" and wait for results
  4. Review findings in the results table and use stat cards to filter
  5. Export results or manage suppressions as needed

üìÅ File Upload

  • Single File: Click "Choose Files" and select one file
  • Multiple Files: Hold Ctrl/Cmd while selecting, or drag multiple files
  • Drag & Drop: Drag files directly onto the upload area
  • Progress Tracking: Shows current file being processed

‚öôÔ∏è Configuration

  • Confidence Levels: Filter by HIGH, MEDIUM, LOW, or all
  • Detection Types: Select specific validators or run all
  • Verbose Output: Show detailed metadata for findings
  • Show Match: Display actual text (otherwise shows [HIDDEN])

üö´ Suppression Management

  • Bulk Operations: Select multiple rules with checkboxes
  • Individual Actions: Enable/disable/edit/remove buttons
  • Undo Support: Undo button appears after operations
  • View Rules: Click "Suppressions" tab

üìä Results Display

  • Export Options: Download as CSV, GitLab SAST, JSON, JUnit, Text, or YAML
  • Interactive Stats: Click stat cards to filter results
  • Real-time Updates: Results appear as files complete
  • Smart Sorting: Multi-level sort by confidence, filename, line
  • Table Sorting: Click column headers to sort

üì§ Export Formats

  • CSV: Spreadsheet-compatible format for analysis
  • GitLab SAST: GitLab Security Report format for Security Dashboard
  • JSON: Structured data for programmatic processing
  • JUnit XML: Test report format for CI/CD integration
  • SARIF: SARIF 2.1.0 format for GitHub Security and other SARIF-compatible tools
  • Text: Human-readable plain text format
  • YAML: Human-readable configuration format

üìÅ Supported File Types

  • Audio: FLAC, M4A, MP3, OGG, WAV
  • Documents: Excel, PDF, PowerPoint, Word
  • Images: BMP, GIF, JPEG, PNG, TIFF, WebP
  • Text: .csv, .json, .log, .txt, .xml, source code
  • Video: AVI, MKV, MOV, MP4, WMV

üéØ Detection Types

  • CREDIT_CARD: Vendor validation, Luhn algorithm (15+ card brands)
  • EMAIL: RFC-compliant validation with domain checks
  • INTELLECTUAL_PROPERTY: Patents, trademarks, copyrights
  • IP_ADDRESS: IPv4 and IPv6 address detection
  • METADATA: EXIF, GPS, document properties
  • PASSPORT: Multi-country formats (US, UK, Canada, EU, MRZ)
  • PERSON_NAME: Human names with pattern matching and name databases
  • PHONE: International and domestic formats
  • SECRETS: API keys, tokens, credentials (40+ patterns)
  • SOCIAL_MEDIA: Platform handles, profiles, and URLs
  • SSN: US Social Security Numbers with area validation
üí° Tip: Use the exact validator names (e.g., CREDIT_CARD, EMAIL) when specifying --checks in CLI commands.

üìä Confidence Levels

  • HIGH (90-100%): Very likely sensitive
  • MEDIUM (60-89%): Possibly sensitive
  • LOW (0-59%): Likely false positive

üîí Security Features

  • Audit Trail: Comprehensive logging
  • Compliance Ready: Data residency support
  • Local Processing: No cloud by default
  • Memory Scrubbing: Secure memory handling
  • No Data Retention: Files cleaned up automatically

üí° Usage Tips

  • Use confidence filtering to reduce false positives
  • Enable verbose mode for detailed metadata
  • Click stat cards to filter by confidence level
  • Click table headers to sort results
  • Review expired suppressions regularly
  • Export results for further analysis

‚å®Ô∏è CLI Command Examples

Scan all validators:
./ferret-scan --file document.txt
Specific validators:
./ferret-scan --file document.txt --checks CREDIT_CARD,EMAIL,PERSON_NAME,PHONE
High confidence only:
./ferret-scan --file document.txt --confidence high --verbose

  • Expiration: Rules expire automatically
